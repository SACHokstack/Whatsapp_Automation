# Course Overview

Modern Software Testing with AI-Assisted Automation & CI/CD Integration is a 4-day hands-on training program (28 hours) that equips participants with modern software testing skills by combining automation frameworks, AI-assisted testing techniques, and CI/CD pipeline integration. HRDC Registration No: 10001697599.

This is the August 2026 intake of the same course, held at Ibis PJCC Petaling Jaya on 17–20 August 2026.

Who should attend:
- QA Engineers / Test Engineers
- Software Developers
- Automation Engineers
- DevOps Engineers
- Technical Leads

Target industries: IT/Software Development, Banking & Financial Services, Telecommunications, Semiconductor & Electronics, Digital Services.

Prerequisites (mandatory):
- Basic understanding of software testing concepts
- Familiarity with web applications
- Basic programming knowledge (JavaScript/TypeScript preferred but not required)

Lab requirements: Laptop with 16 GB RAM, VS Code, Node.js, npm, Chrome/Edge/Firefox browsers, Azure DevOps account, GitHub Copilot access.

Course structure (4 days):
- Day 1: TypeScript fundamentals, modern QA landscape, locators, Playwright introduction
- Day 2: UI automation with Playwright, AI-assisted testing using ChatGPT and GitHub Copilot
- Day 3: Page Object Model framework, API testing with Playwright, data-driven testing, stability & maintainability
- Day 4: CI/CD integration with Azure DevOps, BDD with Cucumber, final hands-on end-to-end project

Key topics:
- Playwright UI automation (from scratch — no prior Playwright experience needed)
- TypeScript: variables, classes, interfaces, control flow, error handling
- AI tools: ChatGPT for test script generation, GitHub Copilot for coding assistance
- Page Object Model (POM): scalable, maintainable framework design
- API testing: REST API validation, combining UI + API automation
- Data-driven testing: parameterization, test data management, environment configs
- Handling flaky tests, smart waits, debugging Playwright scripts
- Azure DevOps CI/CD pipeline: running automated tests, reporting, execution analysis
- BDD with Cucumber: feature files, business requirement alignment
- Final project: end-to-end automation covering UI, API, and pipeline execution

Training approach: 90% hands-on labs, 10% theory. Includes pre-assessment, continuous lab assessments, and post-assessment.

By the end of the course, participants can design, build, and deploy modern automated testing solutions using Playwright, AI tools, API testing, and CI/CD practices.

Investment: RM 6,000 per participant. Group discount available for minimum 3 pax. 100% HRDC claimable.
