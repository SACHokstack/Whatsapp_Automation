# Course Overview

Embedded C Programming and GDB Debugging is a practical 2-day course (14 hours) that gives participants a solid foundation in C programming with a focus on embedded systems development, and teaches hands-on debugging using GDB. HRDC Registration No: 10001676654.

Held at Timmins Training Center, Penang on 30–31 July 2026.

Who should attend:
- Embedded software engineers
- Firmware engineers
- Junior embedded developers
- Software engineers moving into embedded systems
- Fresh graduates entering firmware or embedded roles
- Engineers who want to strengthen C programming and debugging fundamentals

Recommended background:
- Basic programming knowledge (variables, functions, control flow)
- Prior exposure to C programming is helpful
- Basic computer architecture understanding is an advantage

Course structure (4 sessions, 2 days):
- Session 1: C Basics and Storage Classes — data types, type qualifiers, operators, storage classes (auto, static, extern, register)
- Session 2: Pointers and User-Defined Types — referencing/dereferencing, pointer errors, strings, structures, unions
- Session 3: Functions and Preprocessing — modular functions, parameters/return values, command-line arguments, macros, preprocessing directives
- Session 4: Debugging with GDB — GDB basics, setting breakpoints and watchpoints, examining source/data/stack, fixing issues in a real C application

Key topics:
- C data types and sizes (hands-on: check sizes of int, char, float, etc.)
- Pointer manipulation: referencing, dereferencing, common pitfalls
- Structures and unions for user-defined data types
- Modular programming with functions and reusability
- Macro-based preprocessing for conditional compilation
- GDB debugger: breakpoints, watchpoints, variable inspection, call stack analysis
- Fixing real bugs in provided C applications using GDB

Tools used: C, GCC, GDB, Command Line, VS Code, Eclipse.

Practical outcomes:
- Write clearer, more confident C code for embedded applications
- Understand pointers and memory use, avoid common errors
- Debug C programs using GDB breakpoints, watchpoints, and stack inspection
- Build the embedded C foundation needed for firmware and Linux driver development

Investment: RM 3,000 per participant. Group discount available for minimum 3 pax. HRDC claimable.
